## Bun Do's and Don'ts

### DO
- Use bun's built-in test runner
- Leverage native TypeScript support
- Use workspace features for monorepos
- Take advantage of faster installs
- Use bunx instead of npx

### DON'T
- Mix bun with npm/yarn lockfiles
- Assume 100% Node compatibility
- Ignore bun-specific APIs
- Use Node polyfills unnecessarily
- Skip lockfile commits
